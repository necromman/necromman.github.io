# Series Plan — 시리즈 기획 워크플로우

새로운 콘텐츠 시리즈를 제작하기 전에 자료조사와 기획을 먼저 수행하는 선행 워크플로우 스킬.
**콘텐츠 워크플로우(`.claude/rules/content-workflow.md`)의 Phase 0~1을 구체적으로 실행하는 스킬이다.**

## 왜 필요한가

콘텐츠를 바로 HTML로 만들면:
- 자료가 부실해지고
- 시리즈 간 논리 흐름이 끊기고
- 소설이면 설정 모순, 반복 표현, 클리셰가 발생하고
- 나중에 구조를 바꾸려면 전체를 다시 만들어야 한다

기획 문서를 먼저 만들면:
- 자료조사 결과를 한 곳에 정리할 수 있고
- 시리즈 전체 구조를 조감할 수 있고
- 소설이면 세계관·캐릭터·플롯의 일관성을 사전 보장하고
- 콘텐츠 제작 시 기획서를 참조하면 일관성이 유지된다

## 시작: 콘텐츠 유형 판별

**이 스킬이 트리거되면 Orchestrator는 먼저 콘텐츠 유형을 판별한다:**

| 유형 | 판별 기준 | 기획 경로 |
|------|----------|----------|
| **정보전달형** | 정리, 분석, 요약, 해설, 트렌드 | 자료조사 → 시리즈 구조 설계 |
| **소설형** | 소설, 이야기, 창작, 단편, 판타지, SF | 자료조사 → 세계관 → 캐릭터 → 플롯 → 시리즈 구조 |
| **혼합형** | 정보를 소설/서사 형식으로 전달 | 자료조사 → 정보 구조 → 세계관 → 캐릭터 → 플롯 → 통합 구조 |

판별 불가 시 사용자에게 확인한다.

## 폴더 구조

```
plans/
└── [시리즈-슬러그]/
    ├── research.md           # 자료조사 결과 (모든 유형)
    ├── series-plan.md        # 시리즈 기획서 (모든 유형)
    ├── worldbuilding.md      # 세계관 설계서 (소설형/혼합형)
    ├── characters.md         # 캐릭터 설계서 (소설형/혼합형)
    └── plot-outline.md       # 플롯 설계서 (소설형/혼합형)
```

**규칙:**
- `plans/` 폴더는 프로젝트 루트에 위치한다
- 시리즈 슬러그는 `content/` 아래 폴더명과 동일하게 맞춘다
- 정보전달형은 `research.md` + `series-plan.md`만 생성
- 소설형/혼합형은 전체 5개 문서 생성

## Phase 0: 자료조사 (Researcher 에이전트)

웹 검색과 코드베이스 분석을 통해 시리즈 주제에 대한 데이터를 수집한다.

### 필수 조사 항목 (모든 유형)

1. **기존 컨벤션 분석** ★ 항상 수행
   - 프로젝트 내 기존 콘텐츠의 문체, 구조, 패턴 분석
   - `editorial-base.css` 컴포넌트 확인
   - 같은 유형의 기존 시리즈가 있으면 참조
   - **기존 시리즈에서 발생한 문제점 확인** (반복 표현, 클리셰, 사용자 불만)

2. **주제 조사**
   - 정량 데이터 (통계, 수치, 연구 결과)
   - 전문가 의견/인용
   - 한국 맥락의 데이터 (글로벌만으로 부족)
   - 반대 의견/반박 자료 (균형 확보)
   - 출처 URL

### 소설형 추가 조사 항목

3. **장르 분석**
   - 유사 장르 작품의 세계관/캐릭터 패턴
   - 해당 장르의 **클리셰 목록** (회피 대상)
   - 참고할 수 있는 우수 작품 레퍼런스
   - 흔히 빠지는 함정 (설정 모순, 스케일 문제, 시간축 오류)

4. **기존 소설 콘텐츠 문제점 분석**
   - 프로젝트 내 기존 소설 시리즈의 문제점 목록화
   - 반복 표현 패턴 (예: 특정 수치의 과도한 반복)
   - 파워 스케일 문제, 에필로그 공식화, 설명조 대사 등
   - **이번 시리즈에서 반드시 회피할 패턴 목록 작성**

### 자료조사 원칙
- 2024년 이후 최신 자료를 우선한다
- 학술 논문, 공식 리포트 > 블로그, 유튜브
- 한국 데이터가 없으면 글로벌 데이터 + "한국에서는" 맥락 추가
- 하나의 주장에 반드시 근거 데이터를 붙인다

### 출력: `research.md`

```markdown
# 자료조사: [시리즈 제목]

> **조사일:** YYYY-MM-DD
> **콘텐츠 유형:** 정보전달형 / 소설형 / 혼합형

## 1. 기존 컨벤션 분석
- 기존 시리즈 패턴
- 확인된 문제점 / 회피 패턴

## 2. 주제 데이터
- 핵심 데이터와 출처

## 3. [소설] 장르 분석
- 장르 레퍼런스
- 클리셰 목록 (회피 대상)
- 기존 소설 콘텐츠 문제점

## 4. 참고 자료
- 출처 URL 목록
```

## Phase 1-A: 소설형 설계 (World Builder → Character Designer → Story Architect)

소설형/혼합형일 때만 실행한다. 각 에이전트가 순차적으로 설계 문서를 작성한다.

### World Builder: 세계관 설계서 (`worldbuilding.md`)

```markdown
# 세계관 설계서: [시리즈 제목]

## 1. 시공간 설정
- 장소 (지리, 주요 무대)
- 시대 (과거/현재/미래, 시간축)
- 규모 (마을/도시/대륙/우주)

## 2. 규칙 체계
- 마법/기술/사회 시스템의 규칙
- 제약과 대가 (★ 파워에는 반드시 대가가 있어야 함)
- 예외 사항 (있다면 명확히 정의)

## 3. 역사적 배경
- 시리즈 이전에 일어난 주요 사건
- 현재 상황에 영향을 미치는 과거
- 시간축 정리 (연표)

## 4. 핵심 용어 사전 (Glossary)
- 세계관 고유 용어 정의

## 5. 설정 규칙 (금지 사항)
- 이 세계관에서 절대 일어나서는 안 되는 일
- 모순 방지를 위한 명시적 제약
- 반복 방지 규칙 (같은 수치/표현 3회 이상 금지)
```

### Character Designer: 캐릭터 설계서 (`characters.md`)

```markdown
# 캐릭터 설계서: [시리즈 제목]

## 주요 인물

### [캐릭터명]
- **역할**: 주인공 / 조력자 / 적대자
- **외형**: 간결한 외모 묘사
- **성격 핵심**: 3단어로 요약
- **동기**: 무엇을 원하는가
- **약점/결함**: (★ 완벽한 캐릭터 금지)
- **말투 가이드**: 존댓말/반말, 어조, 자주 쓰는 표현, 절대 안 쓰는 표현
- **캐릭터 아크**: 시작 상태 → 변화 계기 → 종착 상태

## 인물 관계도
- A ↔ B: 관계 설명

## 금지 사항 (OOC 방지)
- [캐릭터]는 절대 ~하지 않는다
- [캐릭터]의 말투에 ~는 포함되지 않는다
```

### Story Architect: 플롯 설계서 (`plot-outline.md`)

```markdown
# 플롯 설계서: [시리즈 제목]

## 1. 시리즈 전체 아크
- 기 → 승 → 전 → 결 (또는 적절한 구조)
- 핵심 테마
- 모티프 (반복되는 상징)

## 2. 편별 아웃라인

### 편 1: [제목]
- **핵심 갈등**: 이 편에서 다루는 주요 갈등
- **긴장 수준**: ★☆☆~★★★ (점진적 상승 권장)
- **장면 구성**:
  - Part I: [장면 요약] — 목적
  - Part II: [장면 요약] — 목적
  - Part III: [장면 요약] — 목적
  - 에필로그: [장면 요약] — (★ 매 편 다른 톤의 결말)
- **복선/반전**: 이 편에 심을 복선
- **이전 편 연결**: 이전 편의 어떤 사건이 이 편에 영향을 미치는가

## 3. 긴장 곡선
- 편별 텐션 레벨과 흐름 설계
- 클라이맥스 위치 계획

## 4. 금지 패턴
- 매 편 같은 방식으로 위기 해결 금지 (★★★)
- 에필로그 공식화 금지 (매 편 다른 결말 톤)
- 주인공만의 힘으로 해결하는 패턴 반복 금지
```

## Phase 1-B: 시리즈 기획서 통합 (Planner)

모든 유형에서 실행. 소설형은 위 설계 문서들을 통합한다.

### 출력: `series-plan.md`

```markdown
# 시리즈 기획서: [시리즈 제목]

> **시리즈 슬러그:** `[slug]`
> **시리즈 번호:** Series XX
> **콘텐츠 유형:** 정보전달형 / 소설형 / 혼합형
> **기획일:** YYYY-MM-DD
> **상태:** 기획 완료 → 콘텐츠 제작 대기

## 1. 시리즈 컨셉
- 핵심 질문 / 핵심 서사
- 타겟 독자
- 시리즈 톤

## 2. 자료조사 요약
- 주제별 핵심 데이터와 출처
- [소설] 회피 패턴 목록

## 3. [소설] 설계 요약
- 세계관 핵심 설정 (상세: worldbuilding.md)
- 캐릭터 핵심 프로필 (상세: characters.md)
- 플롯 핵심 흐름 (상세: plot-outline.md)

## 4. 시리즈 구성 (N편)
- 각 편의 슬러그, 제목, 역할, 태그
- Part 구성과 핵심 데이터/장면

## 5. 디자인 노트
- 시리즈 고유 컴포넌트
- editorial-base.css 확장 필요사항

## 6. 제작 순서
- 체크리스트

## 7. 참고 자료 (출처)
- 학술/리포트, 산업 통계, 전문가 의견, 한국 데이터
```

## Phase 1 검수 (사용자 확인 전)

기획서 작성 후 아래 항목을 확인한다:

### 공통 검수
- [ ] 각 편의 핵심 주장에 데이터 근거가 있는가
- [ ] 시리즈 전체가 논리적 흐름을 가지는가
- [ ] 타겟 독자가 명확한가
- [ ] 한국 맥락의 데이터가 포함되어 있는가
- [ ] 슬러그가 영문 소문자 + 하이픈 규칙을 따르는가
- [ ] 출처 URL이 기록되어 있는가

### 소설형 추가 검수
- [ ] 세계관에 모순이 없는가 (worldbuilding.md 내부 정합성)
- [ ] 캐릭터에 약점/결함이 설계되어 있는가
- [ ] 파워에 대가/제약이 명시되어 있는가
- [ ] 편별 위기 해결 방식이 다양한가 (같은 패턴 반복 금지)
- [ ] 에필로그가 편마다 다른 톤인가
- [ ] 특정 표현/수치가 설계서에서조차 과도하게 반복되지 않는가
- [ ] 클리셰 회피 목록이 research.md에 존재하는가

## Phase 2~4: 콘텐츠 워크플로우로 이관

기획서가 완성되면 콘텐츠 워크플로우(`.claude/rules/content-workflow.md`)의 Phase 2(집필) → Phase 3(검증) → Phase 4(배포)로 진행한다.

1. `editorial-content-page` + `seo` 스킬을 참조하여 HTML 제작
2. 기획서의 Part 구성과 핵심 데이터를 그대로 따른다
3. **소설형은 `worldbuilding.md`, `characters.md`, `plot-outline.md`를 참조하며 집필**
4. 제작 중 기획서에 없는 내용이 필요하면 기획서도 함께 업데이트
5. Phase 3 검증 에이전트 투입 (소설: Continuity Checker + Editor / 정보: Fact Checker + Editor)

## 프로젝트 문서 연동

새 시리즈의 기획이 완료되면:
1. `.ai/content/catalog.md`에 시리즈/글 목록 추가
2. `assets/content-data.js`에 시리즈/글 데이터 추가 (랜딩 페이지 반영)
3. `assets/series-nav.js`의 SERIES 데이터에 시리즈 추가
4. `content/index.md` 업데이트 (기록용)
5. `sitemap.xml`에 새 URL 추가
6. 기획서의 `상태`를 "콘텐츠 제작 중" → "완료"로 갱신
